
# A/B-тестирование пуш-уведомлений для удержания оттекающих пользователей (Churning)

## 📌 Бизнес-контекст проекта
Цель эксперимента — оценить эффективность триггерных пуш-уведомлений с персональной скидкой 10% для реактивации сегмента уходящих пользователей ("Churning"). Главная задача — повысить конверсию в покупку, не уронив при этом средний чек (из-за скидки) и не выжигая базу лояльных клиентов агрессивным маркетингом.

## 🛠 Что было сделано (End-to-End дизайн):

1. **Pre-test дизайн эксперимента:**
   * Рассчитан необходимый объем выборки ($N \approx 3800$ пользователей на каждую группу) на основе исторической конверсии и ожидаемого минимального детектируемого эффекта ($MDE = 2\%$) при мощности тестов 80% и уровне значимости $\alpha = 5\%$.
   * Проведена валидация системы сплитования на этапе A/A-тестирования.

2. **Анализ целевой метрики (Conversion Rate):**
   * В тестовой группе (с пушами и скидкой) конверсия статистически значимо выросла с **9.59%** до **12.23%**.
   * Чистый абсолютный прирост составил **+2.64%**, что успешно перекрывает заложенный MDE. Статистическая значимость подтверждена Z-тестом для пропорций ($p\text{-value} = 0.00022$).

3. **Анализ метрики здоровья бизнеса (Average Order Value):**
   <img width="1247" height="518" alt="image" src="https://github.com/user-attachments/assets/d68a52da-6803-4320-a667-4538f5240287" />
   * В ходе разведочного анализа данных ($EDA$) с помощью графиков **QQ-plot** было выявлено лог-нормальное распределение чеков с тяжелыми правыми хвостами (наличие «китов» с аномально крупными покупками).
   * На основе этого был обоснован отказ от классического t-теста Стьюдента. Применен **метод Bootstrap (2000 симуляций)**. 
   * Полученный 95%-ый доверительный интервал для разницы средних составил `[-7.79 €, 36.76 €]`. Так как интервал включает `0`, изменение среднего чека (423.98 € в А против 438.05 € в B) признано **статистически незначимым**. Метрика здоровья бизнеса (Guardrail) защищена.

4. **Анализ контр-метрики негативного эффекта (Unsubscribe Rate):**
   * Для контроля выжигания аудитории был запущен Z-тест на уровень отписок от уведомлений. 
   * Обнаружен сильный побочный эффект: уровень отписок в тестовой группе значимо вырос с **1.51%** до **2.50%** ($p\text{-value} = 0.00198$). Относительный рост негатива составил ~65%.

## 📊 Итоговый продуктовый вывод и рекомендации
Эксперимент выявил классическую бизнес-дилемму: пуш-механика приносит краткосрочный буст конверсии, но при этом **выжигает базу**, лишая компанию бесплатного канала коммуникации с пользователями в будущем.

**Рекомендация для продуктовой команды:** Прямую раскатку фичи на 100% пользователей **заморозить**. Отправить механику на доработку (Итерация №2) для внедрения ограничений по частоте отправки (*Frequency Capping*) и более точечного таргетинга скидки только на чувствительных к цене пользователей.
